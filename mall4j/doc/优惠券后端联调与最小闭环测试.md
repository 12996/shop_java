## 2. Token 准备

## 2.1 管理员 Token（推荐）

优惠券后台接口建议直接使用 dev token 方案（仅本地 dev 环境可用）：

```powershell
$adminTokenResp = Invoke-RestMethod -Method Post -Uri "http://localhost:18085/dev/auth/admin-token"
$adminToken = $adminTokenResp.data.accessToken
```

后续所有后台接口都带：

```text
Authorization: <adminToken>
```

详细限制与排错见：

- [后端自动化验证与调试Token方案.md](/F:/work/project/shop_java/mall4j/doc/后端自动化验证与调试Token方案.md)

## 2.2 用户 Token（前台接口）

`/p/**` 接口必须使用普通用户 Token，不能使用管理员 dev token。

建议做法：

1. 用前台页面正常登录一个测试用户。
2. 从浏览器 Network 或客户端请求头中复制 `Authorization`。
3. 用该 Token 调用 `http://localhost:18086/p/**` 接口。

## 4. 管理员接口验证（创建券、发券）

## 4.1 创建公开领取券（`status=0`）

```powershell
$body = @{
  couponNo = "CPN-PUBLIC-001"
  couponName = "联调公开券-减10"
  reduceAmount = 10.00
  conditionAmount = 99.00

$createPublic = Invoke-RestMethod -Method Post -Uri "http://localhost:18085/admin/coupon" -Headers @{ Authorization = $adminToken } -Body $body -ContentType "application/json"
```

$testUserId = "请替换为真实user_id"

## 4.3 管理员发券（把公开券发给指定用户）

先拿到 `couponId`（可用分页接口查询）：

```powershell
$page = Invoke-RestMethod -Method Get -Uri "http://localhost:18085/admin/coupon/page?pageNum=1&pageSize=20&couponNo=CPN-PUBLIC-001" -Headers @{ Authorization = $adminToken }
$couponId = $page.data.records[0].couponId

$sendBody = @{
  couponId = $couponId
  userId = $testUserId
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri "http://localhost:18085/admin/coupon/send" -Headers @{ Authorization = $adminToken } -Body $sendBody -ContentType "application/json"
```

## 6. 订单链路验证（确认、提交、取消）

## 6.1 确认订单（试算，不改券状态）

说明：

- `couponIds` 只允许 1 张，多于 1 会报错。
- 提交订单依赖确认单缓存，因此必须先走确认接口。
- `addrId=0` 表示取默认地址；如果无默认地址，请改成真实 `addr_id`。

```powershell
$basketId = 1   # 替换为真实可下单 basket_id
$selectedCouponId = 1  # 替换为真实可用 coupon_id

$confirmBody = @{
  addrId = 0
  basketIds = @($basketId)
  couponIds = @($selectedCouponId)
} | ConvertTo-Json

$confirmResp = Invoke-RestMethod -Method Post -Uri "http://localhost:18086/p/order/confirm" -Headers @{ Authorization = $userToken } -Body $confirmBody -ContentType "application/json"
```

验收点：

- 返回 `data.shopCartOrders[].coupons`（含 `canUse`、`unusableReason`）
- 选中券后 `couponReduce > 0`
- `selectedCouponId` 被写入对应店铺订单

## 6.2 提交订单（最终核销）

`shopId` 建议从上一步 `confirmResp.data.shopCartOrders[].shopId` 复制：

```powershell
$shopId = 1  # 替换为确认单返回的 shopId

$submitBody = @{
  orderShopParam = @(
    @{
      shopId = $shopId
      remarks = "coupon-task6"
    }
  )
} | ConvertTo-Json -Depth 4

$submitResp = Invoke-RestMethod -Method Post -Uri "http://localhost:18086/p/order/submit" -Headers @{ Authorization = $userToken } -Body $submitBody -ContentType "application/json"
$orderNumbers = $submitResp.data.orderNumbers
```

验收点：

- 下单成功后，券从 `status=1` 变成 `status=2`
- `tz_coupon.order_number` 写入提交后的订单号
- `tz_coupon.use_time` 非空

## 6.3 取消订单（退券）

取订单号（`$orderNumbers` 可能是逗号分隔，多店铺时按实际拆分）：

```powershell
$firstOrderNumber = $orderNumbers.Split(",")[0]
Invoke-RestMethod -Method Put -Uri "http://localhost:18086/p/myOrder/cancel/$firstOrderNumber" -Headers @{ Authorization = $userToken }
```

验收点：

- 取消成功后，券从 `status=2` 回退到 `status=1`
- `order_number`、`use_time` 被清空

## 7. SQL 验证脚本（建议每步执行）

```sql
-- 按券编号观察状态流转
SELECT coupon_id, coupon_no, user_id, status, order_number, receive_time, use_time, start_time, end_time, update_time
FROM tz_coupon
WHERE coupon_no IN ('CPN-PUBLIC-001', 'CPN-ASSIGN-001')
ORDER BY coupon_id DESC;

-- 按订单号反查券
SELECT coupon_id, coupon_no, status, order_number, use_time
FROM tz_coupon
WHERE order_number = '请替换为提交后的订单号';

-- 订单状态（用于确认取消前后）
SELECT order_id, order_number, user_id, status, actual_total, create_time
FROM tz_order
WHERE order_number = '请替换为提交后的订单号';
```

状态含义：

- `0` 待发放
- `1` 未使用
- `2` 已使用
- `3` 已过期

## 8. 常见问题

- `优惠券不存在或不可领取`：通常是券不是公开领取模式，或不在有效期，或已被领取。
- `一笔订单仅允许使用一张优惠券`：`couponIds` 传了多张。
- `优惠券信息异常，请重新确认订单`：未走确认单直接提交，或确认单缓存已失效。
- `401`：Token 无效、过期，或请求头未带 `Authorization`。
- 管理后台接口 `403`：管理员账号缺少 `admin:coupon:*` 权限点，需要补齐权限数据。
