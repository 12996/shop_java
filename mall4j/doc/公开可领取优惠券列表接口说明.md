# 公开可领取优惠券列表接口说明

## 1. 当前情况

当前用户侧优惠券接口只有：

- `POST /p/coupon/receive/{couponNo}`
- `GET /p/myCoupons`

所以现在没有一条接口可以让前端先获取“公开可领取优惠券列表”。

这意味着：

- 小程序“领券中心”当前没有正确接口可调用
- 商品详情页“领券弹层”当前也没有正确接口可调用

## 2. 现在不该调什么

### 不该调 `GET /p/myCoupons`

原因：

- 这条接口返回的是“当前用户已经拥有的券”
- 它不是“公开可领取券列表”

### 不该调 `POST /p/coupon/receive/{couponNo}`

原因：

- 这条接口只能在前端已经知道 `couponNo` 的前提下调用
- 它不能承担“先拉列表”的职责

## 3. 应补哪条接口

建议新增：

- `GET /p/coupons/public`

建议只返回满足以下条件的券：

- `status = 0`
- `receive_mode = 2`
- `user_id IS NULL`
- `start_time <= now <= end_time`

## 4. 前端正确调用流程

领券中心或商品详情页应按以下顺序调用：

1. 调 `GET /p/coupons/public` 获取公开可领取券列表
2. 用户点击某张券
3. 前端拿该券的 `couponNo`
4. 调 `POST /p/coupon/receive/{couponNo}` 完成领取
5. 如需查看已拥有券，再调 `GET /p/myCoupons`

## 5. 当前阶段说明

这份文档只记录接口缺口与建议方案，不包含代码实现。

如果下一步开始编码，应先按本说明补上“公开可领取优惠券列表接口”，再让前端接入领券中心。
